# 物业管理系统

## How to Run

```bash
# 首次启动（构建镜像并启动）
docker-compose up -d --build

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down

# 停止并清除数据（重新初始化数据库）
docker-compose down -v
```

## 本地开发

### 环境要求
- Node.js 18+
- Java 17+
- Maven 3.8+
- MySQL 8.0+

### 后端启动

```bash
cd backend

# 配置数据库连接（修改 src/main/resources/application.yml）
# spring.datasource.url: jdbc:mysql://localhost:3307/property_management
# spring.datasource.username: root
# spring.datasource.password: your_password

# 编译并启动
mvn spring-boot:run

# 或者打包后运行
mvn clean package -DskipTests
java -jar target/property-management-1.0.0.jar
```

### 前端启动

```bash
cd frontend-admin

# 安装依赖
npm install

# 开发模式启动（默认通过 /api 代理到 http://localhost:8080）
npm run dev

# 如需修改后端地址，编辑 vite.config.js 中的 proxy 配置

# 生产构建
npm run build
```

### 数据库初始化

```bash
# 连接 MySQL 后执行
mysql -u root -p

# 创建数据库并导入表结构
source mysql/schema.sql
source mysql/data.sql

# 可选：导入模拟数据
source mysql/mock-data.sql
```

## 题目内容

使用VUE和数据库编写一个WEB网页程序前端，需求如下： 
一、业主管理： 
1、业主信息：包括业主姓名、楼栋号、单元号、房间号、业主电话、房屋面积，入住时间，房屋状态 
2、车位管理 ：业主可绑定的多个车位号，车位号不可重复。 
3、业主信息和车位信息可使用Excel批量导入。 
二、账单管理： 
1、可以按日期月份自定义生成物业费和车位费账单，账单可以使用A4纸打印或导出。 
2、可以设置定时功能自动生成物业费，可设定收费时间段，超出时间费未缴费为欠费。 
3、可自定义物业费金额与单位，单位为元/平方米/月，可自定义车位管理费金额，格式为元/个/月，增加收费频次功能，内容为周期性和一次性，可自定义单位， 
4、可自定义其他收费项目名称、金额、单位，可自定义收费标准。 
5、可按月、季度、年度、自定义周期生成账单。 
三、缴费管理： 
1、管理员和普通用户都可以进行业主缴费操作。 
2、缴费金额取2位小数。 
3、缴费内容包括：现金、三方债转、POS，缴费内容可自定义添加 
4、缴费区间可自定义，月度、季度、年度。 
5、优惠方案可自定义输入，输入格式为%。 
6、缴费金额根据账单自动带入。 
7、应收账款管理 
各类型费用于每月25日结账，应收账款当期数据于每月25日自动锁定，仅管理员账号可进行解锁及修改； 
每月26日，以房号或车位号为主体，自动生成次月应收账款数据，并计入对应账户的累计数据； 
上述数据生成后，以业主信息为主体，生成对应收费用通知单，通知单需体现费用周期及金额； 
收费通知单应输出为可打印格式，纸张类型为A4。 
四、统计与分析： 
1、可自定义区间生成物业费缴费明细表及汇总表。 
2、可自定义区间生成物业费欠费明细表及汇总表。 
3、可自定义区间生成车位费缴费明细表及汇总表。 
4、可自定义区间生成车位费欠费明细表及汇总表。 
5、分析管理：可自动统计当前收缴率，欠费率，账龄。 
6、以上表单可使用打印机进行打印或导出电子表格。 
7、所有用户可查询全部功能生成的表单，不可修改，仅限查询。 
8、所有仪表盘数据、显示数据均为动态数据，与数据库相连，为实时数据展示 
五、小区布局展示： 
1、以平面图形式展示小区楼栋、车位信息 
2、可自定义小区楼栋、单元、房间号，可显示全部楼号信息，显示入住业主与空置房 
3、可自定义小区车位数量，可显示全部车位号，显示已使用车位与空置车位 
4、小区楼栋与车位可以分2页分开显示 
六、物业管理： 
1、可上传催缴通知单模板、违规通知单模板等，预留上传接口，可将业主信息等带入模板中 
2、模板可进行A4纸打印。 
七、管理工具： 
1、可自定义物业公司名称。 
2、可上传物业公司LOGO。 
3、管理员账户名称为admin，初始密码为123456。管理员可新建普通用户，数量不限。 
4、可新建多套账，第一至第四项功能均可设置功能完整的独立账套。 
5、管理员可进行初始化操作，其他用户不可。 
6、建立表单生成模板功能。 
7、管理员可自定义普通用户权限。 
8、可进行数据备份与恢复。 
9、可进行系统升级迭代功能。 
10.系统可以进行初始化功能。 
11.可以切换浅色和深色模式

## 项目介绍

一套完整的物业管理系统，包含业主管理、车位管理、账单管理、缴费管理、统计分析、小区布局展示等功能。支持深色/浅色主题切换。

## 技术栈

- 前端：Vue 3 + Vite + Element Plus + Pinia + ECharts
- 后端：Java 17 + Spring Boot 3 + MyBatis-Plus + MySQL 8.0
- 部署：Docker + Docker Compose

## 功能模块

1. 仪表盘：业主/车位统计、收费概览、收缴率图表
2. 业主管理：业主信息CRUD、Excel批量导入
3. 车位管理：车位CRUD、业主绑定/解绑、Excel导入
4. 收费标准：物业费、车位费、自定义收费项目配置、缴费方式管理
5. 账单管理：手动/定时生成账单、多周期支持、账单调度配置
6. 缴费管理：多支付方式、优惠方案、缴费记录
7. 应收账款：账期管理、锁定功能、通知单生成
8. 统计分析：缴费/欠费明细、A4打印、Excel导出
9. 小区布局：楼栋平面图、车位平面图可视化
10. 模板管理：催缴通知单、违规通知单模板上传与预览
11. 系统管理（管理员）：用户管理、权限配置、账套管理、系统配置、操作日志、数据备份恢复

## 系统升级说明

当前版本在"系统管理 > 系统配置"页面提供版本号和更新日志展示。

对于更复杂的升级功能（如在线升级、Flyway/Liquibase 数据库迁移脚本管理、静态文件热替换等），需要额外的基础设施支持，建议通过以下方式实现：

1. **Docker 部署升级**：修改 docker-compose.yml 中的镜像版本，执行 `docker-compose up -d --build` 重新构建
2. **数据库迁移**：将 SQL 变更脚本放入 mysql/ 目录，手动执行或通过 CI/CD 流程自动执行
3. **数据备份**：升级前通过"系统管理 > 数据备份"功能导出数据，确保可回滚
